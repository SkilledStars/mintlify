## Generating your API Key

You can generate an API key through your super.fans <a href="https://super.fans/dashboard/developers">developer portal</a>.  All requests to the super.fans API must include an authorization header with your API key as the bearer token.

<CodeGroup>

```bash curl
curl --compressed -H 'Authorization: Bearer <JWT_TOKEN>'  https://super.fans/api/v1/pipes/total_likes.json
```

```javascript fetch
let url = new URL(`https://api.us-east.tinybird.co/v0/pipes/total_likes.json`)


const result = await fetch(url, {
  headers: {
    Authorization: 'Bearer <JWT_TOKEN>'
  }
}) 
  .then(r => r.json())
  .then(r => r)
  .catch(e => e.toString())

if (!result.data) {
  console.error(`there is a problem running the query: ${result}`);
} else {
  console.table(result.data)
  console.log("** Query columns **")
  for (let column of result.meta) {
    console.log(`${column.name} -> ${column.type}`)
  }
}
```
</CodeGroup>

## Making Requests
Most fields will require setting the following fields:
<ResponseField name="keywords" type="Array(string)" required>
The keywords you want to filter by

Example: `[keyword1,keyword2]`

</ResponseField>
<ResponseField name="startDate" type="Date" required>
The start date of the time period to analyze

Example: `1/1/2024`

</ResponseField>
<ResponseField name="endDate" type="Date" required>
The end date of the time period to analyze

Example: `1/30/2024`

</ResponseField>

Each request will contain various other fields depending on the request type and type of filtering.  To see a full list of endpoints available, checkout the <a href="https://docs.super.fans/api-reference">api reference</a>.
